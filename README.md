# ephemeral-image-factory

Terraform configuration for spinning up an ephemeral GPU instance (AWS EC2 g4dn.xlarge / NVIDIA T4) running [Stable Diffusion WebUI Forge](https://github.com/lllyasviel/stable-diffusion-webui-forge). Models are stored in S3 and synced at boot, so the instance can be destroyed and recreated without losing assets.

## Architecture

- **EC2**: g4dn.xlarge (T4 GPU, 16GB VRAM) with Deep Learning AMI
- **Storage**: S3 bucket for models, LoRAs, embeddings, and generated outputs
- **Networking**: VPC with public subnet, security group restricted to your IP only
- **Key pair**: RSA 4096-bit key generated by Terraform, saved locally as `sd-forge-key.pem`
- **IAM**: EC2 instance role with scoped S3 read/write access (no hardcoded credentials)

## Prerequisites

- Terraform >= 1.0.0
- AWS CLI configured (`aws configure`)
- An existing S3 bucket with your models uploaded

## Usage

### 1. Create terraform.tfvars

`terraform.tfvars` is gitignored because it contains your personal IP address. Create it manually:

```hcl
my_ip          = "YOUR_GLOBAL_IP/32"   # e.g. "203.0.113.5/32"
s3_bucket_name = "YOUR_S3_BUCKET_NAME"
```

Your global IP can be checked with:

```bash
curl -s https://checkip.amazonaws.com
```

Alternatively, set environment variables instead of a tfvars file:

```bash
export TF_VAR_my_ip="$(curl -s https://checkip.amazonaws.com)/32"
export TF_VAR_s3_bucket_name="your-bucket-name"
```

### 2. Upload models to S3 beforehand

```bash
aws s3 cp your_model.safetensors s3://YOUR_S3_BUCKET_NAME/models/
aws s3 cp your_lora.safetensors   s3://YOUR_S3_BUCKET_NAME/loras/
```

### 3. Launch

```bash
terraform init
terraform apply
```

The WebUI URL and SSH command are printed as outputs after apply.

### 4. Destroy when done

```bash
terraform destroy
```

The S3 bucket is managed outside Terraform and will not be deleted.

## Cost notes

- g4dn.xlarge on-demand: ~$0.526/hr (us-west-2)
- Destroy the instance when not in use — models persist in S3
